## How to Use

### Option A: Incremental (Recommended)

Build milestone by milestone for better control and testing at each step:

1. Copy `product-plan/` to your codebase
2. Install dependency: `npm install lucide-react`
3. Start with Shell: `instructions/incremental/01-shell.md`
4. For each section:
   - Open `prompts/section-prompt.md`
   - Fill in `SECTION_NAME`, `SECTION_ID`, `NN` at the top
   - Paste into your coding agent (Claude Code, Cursor, Copilot, etc.)
   - Answer the agent's questions, then implement
5. Test after each milestone before proceeding

### Option B: One-Shot

Build the entire app in one session:

1. Copy `product-plan/` to your codebase
2. Open `prompts/one-shot-prompt.md`
3. Paste into your coding agent
4. Answer the clarifying questions about your tech stack and auth
5. Let the agent plan and implement everything

## Key Integration Notes

- **DM Sans font** — Load via Google Fonts (see `design-system/fonts.md`)
- **lucide-react** — Required for `AppShell/UserMenu` icons: `npm install lucide-react`
- **oklch() colors** — All colors use oklch(). Ensure your CSS target supports it (all modern browsers do)
- **UVR verification** — The family sign-in flow fires `onSignIn(email, uvrCode, categoryId)`. You must verify the UVR code against the Ukrainian Military Movement registry on your backend
- **No funds handling** — The donation flow never involves money on the platform side. `onDonate` just logs a pending record; admin approves to count it toward jar totals
- **Partnership emails** — `PartnershipWidget.onSubmit` fires when an org submits inquiry. Wire it to your email provider to send confirmation

## Tips

- Read `product-plan/sections/[section-id]/tests.md` before implementing each section — tests describe the expected behavior precisely
- `sample-data.json` files contain realistic Ukrainian names and stories — use them for development and testing
- The `data-shapes/overview.ts` file gives you all entity types in one place for quick reference
- The components use inline styles with `oklch()` values — no special Tailwind setup required to start
